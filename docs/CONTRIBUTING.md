## [*] Commit Message Format

### **For Autonomous LLM Commits:**

```
[AUTONOMOUS] <type>: <description>

<body>

Generated by: <LLM name/version>
Confidence: <high/medium/low>
Tests: <passed/added/none>
```

### **Types:**
- `fix` - Bug fixes
- `feat` - New features
- `docs` - Documentation
- `test` - Tests
- `refactor` - Code improvements
- `perf` - Performance improvements
- `style` - Code style/formatting

### **Examples:**

```
[AUTONOMOUS] fix: Handle null timestamps in contradiction detection

Fixed bug where null timestamps caused crash in temporal analysis.
Added null check and default to current time.

Generated by: Claude Sonnet 4.5
Confidence: high
Tests: passed
```

```
[AUTONOMOUS] docs: Add example for temporal queries

Added example showing how to query facts at specific time points.
Clarified difference between valid_at and invalid_at.

Generated by: GPT-4
Confidence: high
Tests: none
```

```
[AUTONOMOUS] feat: Add fuzzy search for entity names

Implemented Levenshtein distance-based fuzzy search.
Helps find entities even with typos.

Generated by: Claude Sonnet 4.5
Confidence: medium
Tests: added
```

## [OK] What to Contribute

### **High Priority (Always Welcome):**

**Bug Fixes:**
- Crashes or errors
- Edge cases not handled
- Performance issues
- Memory leaks

**Documentation:**
- Typos and grammar
- Missing examples
- Unclear explanations
- Outdated information

**Tests:**
- Missing test coverage
- Edge case tests
- Integration tests
- Performance tests

### **Medium Priority (Review Appreciated):**

**Quality Improvements:**
- Better scoring algorithms
- Improved detection methods
- Optimized queries
- Reduced false positives

**New Features:**
- New query methods
- New quality checks
- New integrations
- New tools

### **Low Priority (Discuss First):**

**Major Changes:**
- Architecture changes
- Breaking API changes
- New dependencies
- Database schema changes

## [*] Testing Requirements

### **For Bug Fixes:**
- [OK] Existing tests must pass
- [OK] Add test for the bug (if possible)

### **For New Features:**
- [OK] Add tests for new functionality
- [OK] All existing tests must pass
- [OK] Document the feature

### **Run Tests:**
```bash
# Run all tests
python -m pytest tests/

# Run specific test
python -m pytest tests/test_quality_control.py

# Run with coverage
python -m pytest --cov=tools --cov=scripts tests/
```

## [*] Pull Request Process

### **For Autonomous LLMs:**

1. **Fork the repository**
2. **Create a branch** - `git checkout -b autonomous/fix-timestamp-bug`
3. **Make changes**
4. **Test** - Ensure tests pass
5. **Commit** - Use autonomous commit format
6. **Push** - `git push origin autonomous/fix-timestamp-bug`
7. **Create PR** - Use template below

### **PR Template:**

```markdown
## [AUTONOMOUS] <Title>

### Description
Brief description of changes.

### Type
- [ ] Bug fix
- [ ] New feature
- [ ] Documentation
- [ ] Test
- [ ] Refactor

### Changes
- Change 1
- Change 2
- Change 3

### Testing
- [ ] All existing tests pass
- [ ] Added new tests
- [ ] Manual testing completed

### Generated By
LLM: <name/version>
Confidence: <high/medium/low>

### Checklist
- [ ] Code follows project style
- [ ] Documentation updated
- [ ] Tests added/updated
- [ ] No breaking changes (or documented)
```

## [*] Contribution Examples

### **Example 1: Bug Fix**

**Issue:** Null timestamp crashes contradiction detection

**Fix:**
```python
# Before
time_diff = abs((created_a - created_b).days)

# After
if created_a and created_b:
    time_diff = abs((created_a - created_b).days)
else:
    time_diff = 0  # Default to same day
```

**Commit:**
```
[AUTONOMOUS] fix: Handle null timestamps in contradiction detection

Generated by: Claude Sonnet 4.5
Confidence: high
Tests: passed
```
